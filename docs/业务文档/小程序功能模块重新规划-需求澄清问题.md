# 小程序功能模块重新规划 - 需求澄清问题

## 一、核心流程与节点设计

### 问题1：身份激活流程
- **当前情况**：现有小程序有 `auth.html` 登录页面
- **需要确认**：
  - 手机号一键登录是否完全替代现有登录方式？
  - "会员权限已解锁"的具体表现是什么？是否需要显示会员标识或权限说明？
  - 存量用户自动关联旧档案的触发时机：登录时自动关联，还是需要用户确认？

### 问题2：深度评估问卷
- **当前情况**：现有 `health-profile.html` 包含健康档案信息
- **需要确认**：
  - "全量深度问卷"是全新设计还是基于现有健康档案页面改造？
  - 问卷是否需要支持分步保存（用户中途退出后继续）？
  - 问卷完成后是否允许用户修改？修改后是否需要重新生成方案？

### 问题3：方案定制等待期
- **当前情况**：现有 `weight-loss-plan.html` 展示减重方案
- **需要确认**：
  - 5-10分钟等待期间，用户是否可以退出小程序？退出后如何通知用户报告已生成？
  - "引导设置服务偏好"是在等待页面内完成，还是跳转到独立页面？
  - 方案生成后，是否支持用户主动触发重新生成？

### 问题4：服务开启与入群
- **当前情况**：现有 `nutritionist.html` 展示AI营养师信息
- **需要确认**：
  - "并入群"是指跳转到企业微信/微信群，还是小程序内的群聊功能？
  - "镜像看板"的具体含义：是实时同步群内数据到小程序，还是小程序作为数据展示端？
  - 90天服务期的倒计时是否需要在小程序中展示？

## 二、功能组件与数据镜像

### 问题5：AI镜像数据源
- **需要确认**：
  - 群内餐照识别：是实时识别还是批量识别？识别失败如何处理？
  - 体脂秤蓝牙连接：是否需要用户手动配对，还是自动识别？
  - 数据同步频率：实时同步还是定时同步（如每小时）？

### 问题6：211饮食结构图
- **当前情况**：现有 `home.html` 有饮食结构展示
- **需要确认**：
  - 211比例是基于单日数据还是多日平均值？
  - 如果用户未在群内发餐照，是否显示默认数据或提示？
  - 是否需要支持查看历史211结构变化趋势？

### 问题7：补剂计划单
- **需要确认**：
  - 补剂计划单是只读展示，还是支持用户标记"已服用"？
  - 是否需要提醒功能（如每日服用提醒）？
  - 补剂计划是否支持根据用户反馈调整（如出现副作用）？

### 问题8：周报复盘入口
- **需要确认**：
  - 周报生成时机：每周固定时间（如周日）还是服务开启后第7天？
  - MVP餐食（红榜餐食）的评选标准是什么？
  - 周报是否支持分享功能？

## 三、第三方系统集成

### 问题9：有赞系统集成
- **需要确认**：
  - "我的课程"、"抗炎百科"、"商城"跳转到有赞小程序的具体实现方式：

  - 课程学习进度是否需要同步回小程序？

### 问题10：服务偏好与AI交互
- **当前情况**：现有 `service-preferences.html` 有服务偏好设置
- **需要确认**：
  - "提醒频次"的具体选项是什么？（如：每日1次、每日3次、每周1次）
  - "服务时段"是指AI文慧在群内的活跃时段，还是用户希望接收提醒的时段？
  - 服务偏好修改后，AI文慧的调整是实时生效还是需要重新计算？
